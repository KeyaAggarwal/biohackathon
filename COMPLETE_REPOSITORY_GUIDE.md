# SynthShield: Complete Repository Guide

## 🎯 What is SynthShield?

**SynthShield** is an AI-powered security system that detects dangerous DNA sequences in synthesis orders. Think of it like airport security for DNA: it scans every order to block attempts to synthesize bioweapons, toxins, or pathogens.

### The Problem We Solve

DNA synthesis is now cheap and accessible. Bad actors could order DNA for:
- Anthrax toxin
- Botulinum toxin
- Ebola virus
- Other pathogens

DNA synthesis providers like Twist and Zymergen try to screen orders, but attackers use tricks:
- **Reverse complement** (hide sequence by flipping it)
- **Frame shifting** (split into pieces with junk DNA)
- **Codon optimization** (change DNA codons while keeping protein the same)
- **Split orders** (buy from multiple companies)
- **Reassembly** (order fragments, assemble into weapon)

SynthShield **catches all these tricks**.

## 🏗️ Architecture: 6 Stages

```
DNA Order → [1] AI Screen → [2] Log → [3] Detect Evasion → [4] Aggregate
                                                              ↓
                                                    [5] L2 Anchor (blockchain)
                                                              ↓
                                                    [6] Hardware Interlock
                                                         (close valve)
```

### Stage 1: AI Screening (Real-time)
**File:** `synthshield/core/sentinel_head.py`

Uses ESM-2 (protein language model) to analyze DNA sequences:
```python
dna_sequence = "ATCGATCG..."
embedding = esm_model.embed(dna_sequence)  # 1280D vector
risk_score = sentinel_head(embedding)      # 0.0 = safe, 1.0 = dangerous
```

**How it works:**
1. Convert DNA → Protein (genetic code)
2. Run through ESM-2 to get embeddings
3. Feed to small neural network (ResNet MLP)
4. Output risk score (0-1)

**Result:** `APPROVED` (< 0.5) or `BLOCKED` (≥ 0.5)

### Stage 2: Cryptographic Logging (Immutable Chain)
**File:** `synthshield/hardware/blackbox.py`

Every decision is logged with HMAC chaining:
```python
Event: {
  "timestamp": "2024-11-20T10:00:00Z",
  "sequence": "ATCGATCG...",
  "risk_score": 0.87,
  "decision": "BLOCKED",
  "previous_hash": "abc123...",
  "hmac": "xyz789..."  # Prevents tampering
}

Chain: E1 → E2 → E3 → ... (each event includes hash of previous)
```

**Why:** Cannot be faked. Every event hashes the previous one. To fake an old decision, you'd need to re-hash entire chain.

### Stage 3: Attack Detection (Evasion Defense)
**Files:** 
- `synthshield/hardware/edison_window.py` (split-order detection)
- `synthshield/core/evasion_detection.py` (reverse complement, frame shift, etc.)
- `synthshield/core/enhanced_edison.py` (integrated defense)

Detects attacks across multiple orders:

**Split-Order Detection (Edison Guard):**
- Maintains rolling buffer of 50,000 base pairs
- When gets 10,000 bp from one customer → checks if it reassembles into dangerous sequence
- Uses temporal analysis (when were orders placed?)

**Evasion Detection (5 types):**
1. **Reverse Complement:** Try reverse-complement version of sequence
2. **Frame Shifting:** Check if hiding in different reading frames
3. **Junk Interleaving:** Remove junk sequences and re-screen
4. **Codon Optimization:** Translate protein, check against protein database
5. **Synthetic Patterns:** Analyze unusual codon usage

### Stage 4: Daily Aggregation
**File:** `synthshield/hardware/blackbox.py`

Merkle tree aggregation of all day's decisions:
```
Day 1 Events: E1, E2, E3, E4, E5

Merkle Tree:
         Root: 0xABC123...
        /            \
    0x111...      0x222...
   /        \    /       \
 H(E1)  H(E2) H(E3)  H(E4)  H(E5)

Benefits:
- Compact representation (one hash for whole day)
- Can verify individual event without showing all others
- Efficient storage
```

### Stage 5: L2 Anchoring (Blockchain)
**File:** `synthshield/web/ethereum_anchor.py`

Submit daily Merkle root to Ethereum L2 (Arbitrum):
```solidity
// On-chain record
contract SynthShield {
    mapping(uint256 day => bytes32 root) public dailyRoots;
    
    function submitDaily(bytes32 root) {
        dailyRoots[today()] = root;
    }
}

Benefit: Immutable permanent record. Can't be deleted/modified later.
```

### Stage 6: Hardware Interlock (Physical Control)
**File:** `synthshield/hardware/interlock.py`

If dangerous sequence detected, physically close valve on synthesizer:
```python
Token = HMAC("CLOSE_VALVE", secret_key)

Synthesizer receives: {
    "action": "close_valve",
    "token": "abc123..."  # Can only be generated by authorized system
}

Hardware: Solenoid valve opens/closes based on token
```

**Why:** Hardware is air-gapped. Can't be hacked remotely.

## 🔄 Data Flow: Complete Example

### User Order: "ATCGATCGATCG..."

```
1. SCREENING LAYER
   Input: "ATCGATCGATCG..."
   ↓
   sentinel_head.py:
   - Convert to protein
   - Get ESM-2 embedding (1280D)
   - Feed to ResNet MLP
   - Output: risk_score = 0.73
   ↓
   screening.py:
   - If risk_score ≥ 0.5: decision = "BLOCKED"
   - If risk_score < 0.5: decision = "APPROVED"
   ↓
   Decision: BLOCKED (0.73 > 0.5)

2. LOGGING LAYER
   blackbox.py:
   - Create event:
     {
       timestamp: 2024-11-20 10:00:00,
       sequence: "ATCGATCGATCG...",
       risk_score: 0.73,
       decision: "BLOCKED",
       previous_hash: "xyz789...",
       hmac: hash(previous_hash + this_data + secret_key)
     }
   - Add to chain (can't be modified without breaking chain)

3. EVASION DETECTION
   evasion_detection.py:
   - Check reverse complement: "CGAT..." → re-screen
   - Check frame shifts: Different reading frames
   - Check junk interleaving: Remove junk, re-screen
   - Check codon optimization: Translate to protein
   - Check synthetic patterns: Analyze codon usage
   ↓
   Result: evasion_detected = True, attacks = ["reverse_complement"]

4. SPLIT-ORDER DETECTION
   edison_window.py:
   - New 50,000 bp from this customer?
   - Add to rolling buffer
   - If total ≥ 10,000 bp: try reassembly
   - Can these fragments reassemble into known toxin?
   ↓
   Result: reassembly_detected = False

5. DAILY AGGREGATION
   At end of day, blackbox.py:
   - All 1000 events from day
   - Build Merkle tree: hash(hash(hash(E1+E2)+hash(E3+E4))+...)
   - Result: daily_root = 0xABC123...

6. L2 ANCHORING
   ethereum_anchor.py:
   - Submit to Arbitrum L2: submitDaily(0xABC123...)
   - Cost: ~$0.05 (cheap!)
   - Benefit: Immutable record forever

7. HARDWARE INTERLOCK
   interlock.py:
   - Generate token: HMAC("CLOSE_VALVE", secret_key)
   - Send to synthesizer hardware
   - Solenoid valve closes
   ↓
   Result: Order BLOCKED. Nothing synthesized.
```

## 🎓 Key Concepts Explained

### ESM-2 (Evolutionary Scale Model)

**What:** Large language model for proteins (like ChatGPT for proteins)

**Why:** Proteins are more fundamental than DNA
- Different DNA can code for same protein (code redundancy)
- ESM-2 captures "protein function" not just DNA sequence
- Harder to evade (can't just change codons)

**How:** 
```
DNA: ATCGATCGATCG...
     ↓
Protein: IDDA... (3-letter code: Isoleucine-Aspartate-Aspartate-Alanine)
     ↓
ESM-2 Embedding: [0.23, -0.15, 0.89, ...] (1280 numbers)
     ↓
Risk Score: 0.73 (from ResNet MLP)
```

### HMAC Chaining

**What:** Hash-based Message Authentication Code

**Why:** 
- Hash prevents tampering (change one bit = completely different hash)
- HMAC requires secret key (can't forge without it)
- Chain (each event includes previous hash) prevents reordering

**How:**
```
Event 1: HMAC("hash_prev=init + data=ATCGATCG...", secret_key) = abc123
Event 2: HMAC("hash_prev=abc123 + data=GCGCGC...", secret_key) = def456
Event 3: HMAC("hash_prev=def456 + data=TATA...", secret_key) = ghi789

To fake Event 2:
- Must change data
- But then HMAC doesn't match
- So Event 3 hash breaks too
- Entire chain breaks
```

### Merkle Trees

**What:** Tree of hashes for efficient aggregation

**Why:** Can prove one event was in the tree without showing all events

**How:**
```
        Root: hash(hash(H1+H2) + hash(H3+H4))
       /                              \
   hash(H1+H2)                    hash(H3+H4)
   /         \                    /         \
  H1         H2                  H3         H4
```

### Ethereum L2 Anchoring

**What:** Submit daily hash to blockchain

**Why:** 
- Immutable (can't change past records)
- Decentralized (not controlled by one company)
- Public ledger (anyone can audit)

**Cost:** ~$0.05 per day (much cheaper than Layer 1)

## 🔒 Security Considerations

### Defense in Depth

We don't rely on **one** thing being secure. Multiple layers:

1. **AI Model** (ESM-2) - Catches semantic similarity
2. **Cryptography** (HMAC chaining) - Prevents tampering
3. **Hardware** (solenoid valve) - Physical control
4. **Blockchain** (Ethereum L2) - Distributed ledger
5. **Temporal** (Edison window) - Detects split orders across time

If one fails, others still work.

### Threat Model

**We protect against:**
- ✅ Single order with dangerous sequence
- ✅ Reverse complement evasion
- ✅ Codon optimization evasion
- ✅ Frame shifting attacks
- ✅ Split orders (buy from multiple providers)
- ✅ Tampering with logs

**We don't protect against:**
- ❌ Insider attack (rogue employee)
- ❌ Supply chain compromise (fake synthesizer)
- ❌ Quantum computing breaking HMAC (far future)

### Keys & Secrets

Store securely:
- Secret key for HMAC chaining (rotation yearly)
- Ethereum private key (never share!)
- API keys (use environment variables, never commit)

See `.env.example` for setup.

## ❓ Common Questions

**Q: Is this actually secure?**
A: It's as secure as current ML + cryptography + hardware can be. Multiple layers means if one fails, others catch it. But perfect security doesn't exist.

**Q: Can it be evaded?**
A: We handle 5 known evasion techniques. New attack vectors will emerge - this is an arms race.

**Q: How fast is it?**
A: AI screening is <10ms per sequence. Evasion detection is ~100ms. Split-order detection is continuous.

**Q: Can I use it commercially?**
A: Apache 2.0 license means yes! You must provide attribution and can't hold us liable.

**Q: Does it work offline?**
A: Yes, except L2 anchoring (needs internet once per day).

**Q: How much does it cost to run?**
A: ~$18/year for blockchain + server costs depend on your setup.
